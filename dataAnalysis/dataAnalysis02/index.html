    数据预先处理

    将数据进行从原始数据 -> 可用数据的转换。data pre-processing 有时候也被称为数据清洗（data cleaning）或data wrangling。

    • handle missing value 找出并处理遗失的数据
    • data formating 统一数据格式
    • data normalization 标准化数据 (centering / scaling) 确定使用的数据范畴
    • data binning 数据分组

    处理遗失的数据

    通常来说遗失的数据表现形式为 “?” “N?A” 0 或者空白。有以下几种处理方式 1. 查看数据的数据源，确保源头数据的完整性和及时性 2. 移除数据（移除单个数据 或 移除整个变量） 3. （使用平均值/mode值）替换遗失的数据，但该种做法会降低数据的准确性

    删除遗失值

    在 python 中可以使用dataframes.dropna() 方法。 * axis=0 删除出现空白值的整行 * axis=1 删除出现空白值的整列。

    一个示例为 df.dropna(subset=["price'], axis=0, inplace=True) 。

    替换遗失值

    可以使用 df.replace(missing_value, new_value)。现在假设我们想要用该列的平均值来替换掉出现的遗失值。 

    # 计算出该列的平均值
mean = df["normalized-losses"].mean()
# 限定作用范畴的列，再使用 df.replace() 方法
df["normalized-losses"].replace(np.nan, mean)

    统一、合理的数据格式 Data Standardization

    数据通常由多人从不同渠道收集，并且以多样的形式保存。统一数据格式会为之后的数据分析带来很大的便利。在很多金科企业和金控公司中也在不断提倡统一数据格式，也称作数据治理。

    以上的示例展示了数据-人员的城市可以表现为 “New York” “NY” 或者 “N.Y”，在这种场景下需要统一其数据格式。

    还有一种场景是将数据转换为最易读、最合理、最符合场景的单元。如摄氏度和华氏度，公里和英里。在决定使用什么单位的时候可以考虑会阅读这份报告、参与数据分析人员的惯例和报告用途。假设我们现在需要将汽车的 mpg 转换为 /100km。在 python 中可以使用 

    # 调整数据格式
df["city-mpg"] = 235/["city-mpg"]
# 重命名列名，可以看到 columns 变量的赋值有点类似于词典
# rename 一般使用两个参数 参数1为columns 参数2为inplace
df.rename(columns={"city_mpg":"city-L/100kn"}, inplace=True)

    在这里需要引进 pandas 中的数据类型。有 objects (“hello”), int64 (1,3) 和 Float64 (3.1415)。有时候当我们调整数据类型时会发生数据类型和调整后的数据不匹配的情况。

    • 使用 df.dtypes() 可以查询数据类型
    • 使用 df.astype() 可以转换数据类型，如df["price"] = df["price].astype("float")

    标准化数据比例 Data Normalization

    想象一个例子，一个测试收集了被试者的年龄和收入，年龄的范畴会在10-100间波动，但是收入的波动范畴会远远大于年龄，在2,000-200,000波动。这样收入对结果的影响会远远大于年龄的对结果的影响，即结果会有偏差。

    有以下几种方式可以解决数据比例的问题 1. Simple feature scaling: 即使用个体值/最大值 2. Min-Max: 即使用 (个体值-最小值) / (最大值-最小值) 3. Z-score: 即使用 (个体值-平均值)/方差 Z- score通常在-3到3之间波动

    数据分组 Binning

    数据分组是将数据分类到不同的类的行为，通过分组，可以将数据完从 numeric -> categorical 的转变。最常见的一个例子是grade。分数 90 以上归类为 A，分数 80 以上归类为 B。通过归类将原先 100 等分的成绩归类到了 A,B,C,D,F 这五个等级。

    # 划分为 3 块，有 4 条线
bins = np.linspace(min(df["pirce"]),max(df["price"]),4)
# 为每个等级取名
group_names = ["low", "medium", "high"]
# 创建新列
df["p-binned"] = pd.cut(df["price"], bins, labels=group_names, include_lowest=True)

    Categorical -> Numeric

    那么假设我们需要把性别 男/女 分别转化为 1/0 呢？在很多数据建模和数据分析中，数字都比类别更好处理。使用 pandas 中的 get_dummies() 可以方便地将类别转换为数字。dummy variable 指只有 1 or 0 两种赋值的变量。

    pd.get_dummies(df["fuel"])

    Python 中实际数据处理

    Python 实际的数据预先处理可以分为这么几个步骤 1. Handle missing data * replace missing data 使用.replace()替换丢失的数据，可以使用 mean 或 mode * delete missing data 使用dropna() 删除具有遗失数据的行或列 3. Data standardization 数据标准化 4. Data normalization 数据正常化 * standardize scale 标准刻度比率 - 对列进行处理 * binning 数据分组 * category -> dummy

    在找出遗失数据的过程中，可以有效使用 .isnull() 来获取一个新的 dataframe 展示每个数据 存在/遗失状态。

    其中，::数据分组:: 先用 bins = np.linspace(min, max, lineCount) 来划分出范畴，如果需要划分 n 个区域则需要 n+1 条线。之后使用新的 list 变量创建每个组的名称。新建 dataframe pd.cut(columns, bins, labels=val, include-lowest=True 来存储分完组的数据。

    ::Category -> dummy:: 可以使用 pd.get_dummies(column) 来创建新的dataframe。然后在通过 .rename() 把列名注释地更清楚一些。之后使用 df = pd.concat([df, newdf], axis=1) 把新生成的列增添进去，然后.drop() 删除旧的、不需要的列。
