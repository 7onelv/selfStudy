    Python05 模型评估和优化

    本章节会涉及到： 1. In-sample data (training data) vs. out-of-sample data (test set) 2. Over-fitting, Under-fitting and Model Selection 过拟合, 欠拟合和模型选择 3. Ridge Regression 岭回归

    数据分组和交叉检验

    可行的方法是将数据分为训练组和测试组。训练组一般为 70% 的数据，而测试组一般为 30% 的数据。使用训练组来培训模型，测试组来测试模型的准确性和适配性表现。当确定最终使用的模型后，应使用全部的数据来培训模型。

    在 python 中可以使用 train_test_split() 来将数据随机分为训练组和测试组。

    from sklearn.model_select import train_test_split
# 数据分组
# random_state is the number generator used for random sampling
x_train, x_test, y_train, y_test = train_test_split(x_data, y_data, test_size=0.3, random_state=0)

    上图左侧是训练组实际和估测值的对比，右侧是测试组实际和估测值的对比。两者之间有较大的一个差距，统计学中将其归纳为泛化误差 / Generalization Error。即 * 训练组的数据样本越大，则每次模型得出的结果差异会更大，但模型结果会更接近真实结果 * 训练组的数据样本越小，则每次模型得出的结果差异会更小，但模型结果会更远离真实结果

    可以使用 ::交叉校验 / Cross Validation:: 来解决泛化误差。交叉校验的概念可以以一个简单的例子来说明。假设数据被分为了 4 组。每次用 3 组训练，1 组测试。总共训练模型 4 次，每组都会成为 3 次训练组， 1 次测试组，最后会生成 4 个 r-squared 值。

    测试交叉检验模型的 R-square 可以使用 cross_val_score() 方法。

    from sklearn.model_selection import cross_val_score, cross_val_predict

# cv 指代将数据分为几组（folds）会返回 R-squared 的平均值
scores = cross_val_score(lr, x_data, y_data, cv=3)
np.mean(scores)
# 交叉检验模型的预测结果 会返回预测值的平均值
Yhat = cross_val_predict(lr, x_data, y_data, cv=3)

    过拟合、欠拟合和模型选择

    • 右侧图示为欠拟合（under-fitting) 即模型过于简单导致模型结果和真实数据差距很大
    • 左侧图示为过拟合（over-fitting)。此时模型 order = 16。过拟合指代模型过于复杂，导致部分区间因为训练样本过少而模型为了贴近训练数据，模型结果出现较大的偏差

    可以看出，随着 order 增多，模型越复杂且模型在训练样本上得出的结果越准确，但随之而来的还有降低的测试样本结果准确性。我们的目标是选择测试样本误差最小的模型，在上图示例中为 order = 8。如果 order < 8，模型欠拟合。如果 order > 8，则模型过拟合。

    然而即使我们选择了 order = 8 的模型，可以看到 MSE 依旧存在。其根本原因是因为数据存在 noise。noise 也被称为 irreducible error，即无法消除的误差。

    在选择使用何种模型的时候，可以考虑使用 order vs. R-squared。上图可以看到当 order = 3 的时候 r-squared 值最高，即模型解释了最高占比的测试数据。而当 order > 3的时候，r-squared 迅速下降。

    Rsqu_test = []
order = [1,2,3,4]
for n in order:
    pr = PolynomialFeatures(degree=n)
    x_train_pr = pr.fit_transform(x_tain[['horsepower']])
    x_test_pr = pr.fit_transform(x_test[['horsepower']])
    lr.fit(x_train_pr, y_train)
    # 计算 r-squared 
    Rsqu_test.append(lr.score(x_test_pr, y_test))

    岭回归 Ridge Regression

    Ridge regression is a regression that is employed in a Multiple regression model when Multicollinearity occurs. Multicollinearity is when there is a strong relationship among the independent variables. Ridge regression is very common with polynomial regression. The next video shows how Ridge regression is used to regularize and reduce the standard errors to avoid over-fitting a regression model

    当多元线性回归中的 x-variable 即独立变量之间存在关系的时候（即多重共线性），岭回归可以有效的解决模型过拟合的问题。多项式回归模型中经常使用岭回归。

    现在假设一个情景，模型已经被训练的符合 99% 的训练数据了。但是有 1% 的数据是 outlier。为了契准这些 outlier 数据，模型的 order 会上到10，如 Yhat = 1 + 2x - 3x^2 - 2x^3 - 12x^4 - 40x^5 + 80x^6 + 71x^7 - 141x^8 - 38x^9 + 75x^10。

    可以看到上述模型的 coef 浮动非常大。最小为1，最大为 141，这也会导致模型在部分 x-varible 区间会失灵。在此时可以引入 Alpha 来解决系数浮动过大的问题。

    Alpha 的选择需要谨慎。

    • 随着 Alpha 的逐渐增大，系数和系数的浮动区域会逐渐变小。但这也会导致模型 under fitting
    • 随着 Alpha 的逐渐变小，系数和系数的浮动区域会逐渐变大，但这会导致模型的 over-fitting

    在决定 Alpha 系数的选择上，可以使用交叉检验。将数据分组，一组分为训练组，另一组分为 predict 组，之后尝试不同的 alpha 值，然后得出一张 alpha -> R-squared 的对应表单值，最后决定应该使用哪个 alpha 值。

    from sklearn.linear_model import Ridge
RidgeModel = Ridge(alpha=0.1)
RidgeModel.fit(X,y)
# R-squared
RidgeModel.score(X,y)
Yhat = RidgeModel.predict(X)

    网格搜索 Grid Search

    岭回归中的 alpha 被称为 hyperparameter 超参数。Scikit-learn 中可以通过网格搜索实现自动化尝试各种 alpha 参数组合，并使用交叉校验来核算 MSE 以验证每种组合的准确性，最后返回最佳的 alpha 系数。

    数据会分为 3 组：Training Set, Validation Set, Test Set. 使用不同的 hyperparameter 和训练数据来训练模型，之后使用验证数据来得出 MSE 或 R-squared 的值，选择 MSE 最小或 R-squared 最大情景下的 hyperparameter 值。最后使用实验组数据来测试模型的准确性。

    from sklearn.linear_model import Ridge
from sklearn.model_selection import GridSearchCV
parameters1 = [{'alpha':[1,10,100,1000,10000,100000],'normalize':[True,False]}]
RR = Ridge()
Grid1 = GridSearchCV(RR, parameters1, cv=4)
Grid1.fit(x_data[['horsepower','curb-weight','highway-mpg']],y_data)
Grid1.best_estimator_

scores = Grid1.cv_results_
scores['mean_test_score']
